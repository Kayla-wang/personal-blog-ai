# System Prompt 设计

> 如何写出高效的系统提示词

## 概述

System Prompt 是定义 AI 行为的核心。好的 System Prompt 能显著提升输出质量、减少幻觉、控制格式。

## 核心概念

### 1. System Prompt 的作用

```python
messages = [
    {
        "role": "system",
        "content": "你是一个专业的代码审查助手..."  # 这就是 System Prompt
    },
    {"role": "user", "content": "帮我审查这段代码"}
]
```

**核心功能**：
- 定义角色身份
- 设定行为规则
- 指定输出格式
- 提供背景知识

### 2. 基本结构

```markdown
# 角色定义
你是 [角色名]，专注于 [领域]。

# 核心任务
你的主要任务是：
1. [任务1]
2. [任务2]

# 行为规则
- [规则1]
- [规则2]

# 输出格式
请按以下格式输出：
[格式说明]

# 示例（可选）
输入：...
输出：...
```

```markdown
# 代码审查助手

你是一个资深代码审查专家，专注于发现代码中的问题和改进点。

## 审查维度
1. 代码质量：可读性、命名、结构

3. 性能问题：算法复杂度、资源泄漏
4. 安全隐患：注入、XSS、敏感信息

## 输出规则
- 先给出整体评价（1-10 分）
- 列出具体问题，按严重程度排序
- 每个问题给出修改建议
- 如果代码没问题，直接说"代码质量良好"

## 注意事项
- 保持客观专业
- 不要过度挑剔格式细节
- 聚焦于真正重要的问题
```

## 实践要点

### 设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| 明确角色 | 清晰定义身份 | "你是数据分析师" vs "你是助手" |
| 具体指令 | 避免模糊表述 | "列出3点" vs "分析一下" |
| 正面表述 | 说要做什么 | "保持简洁" vs "不要啰嗦" |
| 格式约束 | 明确输出结构 | "用 JSON 格式输出" |

### 常见模式

```python
# 1. 专家角色
EXPERT_PROMPT = """
你是一个拥有 10 年经验的 Python 后端工程师。
你熟悉 FastAPI、SQLAlchemy、异步编程。
回答问题时，给出专业、简洁的建议。
"""

# 2. 助手角色 + 约束
ASSISTANT_PROMPT = """
你是用户的编程助手。

规则：
- 先理解问题再回答
- 代码要有注释
- 如果不确定，说明你的假设
- 不要编造不存在的库或 API
"""

# 3. 任务型（结构化输出）
TASK_PROMPT = """
你是一个信息提取工具。从用户输入中提取结构化信息。

输出格式（JSON）：
{
  "name": "姓名",
  "email": "邮箱",
  "intent": "用户意图"
}

如果信息缺失，对应字段填 null。
"""
```

### Token 优化

```python
# ❌ 冗长
system = """
你是一个非常有帮助的人工智能助手，你的目标是尽可能地帮助用户解决他们遇到的各种问题。
你应该保持友善、专业的态度，并且尽量给出详细、准确的回答。
如果你不知道答案，请诚实地告诉用户你不知道，不要编造信息。
"""

# ✅ 精简
system = """
你是编程助手。简洁回答。不确定时说明。
"""
```

## 常见问题

### Q: System Prompt 放多少内容合适？

**简单任务**：2-5 行
**复杂任务**：不超过 500 tokens
**规则多时**：考虑分模块，用标题结构化

### Q: 如何测试 System Prompt 效果？

1. 准备 10-20 个测试 Case
2. 比较不同版本的输出
3. 关注：准确性、格式遵循、边界情况

### Q: System Prompt 对不同模型效果一样吗？

不一样。Claude 对长 Prompt 理解更好，GPT 对结构化指令更敏感。需要针对模型调优。
