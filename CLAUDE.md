## 项目概述

基于 Docusaurus 3.10 构建的个人学习笔记博客，使用 React 19，Node >= 20。支持本地中英文搜索，集成 AI 聊天助手。博客（blog）功能已禁用，站点只有「学习笔记」和「项目实践」两类内容。

## 常用命令

```bash
# 前端（仓库根目录）
npm install     # 安装依赖
npm run start   # 启动开发服务器 (localhost:3000)
npm run build   # 生产构建，输出到 build/
npm run serve   # 预览生产构建
npm run clear   # 清除 .docusaurus 缓存（配置或内容异常时先跑这个）

# API 后端（server/ 目录，独立依赖树）
cd server
npm install
npm run dev     # node --watch，监听 :3001
npm start       # 生产启动
```

## 架构

### 内容与路由（两个独立的 docs 插件）

站点有两套并列的文档内容，各自有独立的路由和自动生成侧边栏：

- **学习笔记** — preset-classic 的默认 docs 插件。内容在 `docs/`，路由 `/notes`（注意 `routeBasePath: 'notes'`，不是默认的 `/docs`），侧边栏配置 `sidebars.js`（sidebarId `tutorialSidebar`）。
- **项目实践** — 额外注册的 `@docusaurus/plugin-content-docs` 实例（id `projects`）。内容在 `projects/`，路由 `/projects`，侧边栏配置 `sidebarsProjects.js`。

两个侧边栏都用 `autogenerated`，从对应目录结构自动生成，无需手动维护条目。要新增内容分类，在 `docs/` 或 `projects/` 下建文件夹即可。

### 搜索

`@easyops-cn/docusaurus-search-local`（配置在 `docusaurus.config.js` 的 `themes`）。索引 `/notes` 和 `/projects` 两个路由（`docsRouteBasePath`），不索引 blog。改动内容路由时需同步更新此处的 `docsRouteBasePath`。

### AI 聊天

- 前端组件 `src/components/AIChatWidget/`（悬浮窗，用 `@ai-sdk/react` 的 `useChat`），通过 `src/theme/Root.js` 全局注入。请求发往 `/api/chat`。
- **存在两套后端实现**，对应不同部署方式：
  - `server/`（Express + PM2，用于阿里云自托管）：入口 `server/index.js` 监听 :3001，路由 `server/routes/chat.js` 手动写 SSE 流。用 OpenAI 兼容接口，模型/密钥来自 `server/.env`（`AI_BASE_URL` / `AI_API_KEY` / `AI_MODEL`）。此目录用较旧的 AI SDK v4，与根目录的 v6 不是同一份依赖。
  - `api/chat.js`（Vercel Edge Function）：硬编码通义千问（dashscope），密钥 `DASHSCOPE_API_KEY`，用 `toDataStreamResponse()`。
- 生产环境需由 Nginx / 部署平台把 `/api/chat` 反代到对应后端，前端才能连通。

## 部署

推送到 `main` 分支触发 GitHub Actions（`.github/workflows/deploy.yml`），通过 SSH 部署到阿里云服务器，分两步：
1. `npm run build` 后用 rsync 把 `build/` 同步到 `/var/www/blog`（静态站）。
2. SSH 进服务器重启 `/var/www/blog-api` 的 API 服务（`pm2 reload blog-api`，配置见 `server/ecosystem.config.cjs`）。

**GitHub Secrets:** `SSH_PRIVATE_KEY`、`REMOTE_HOST`、`REMOTE_USER`。

服务器辅助文件：`deploy/nginx.conf`、`deploy/setup-server.sh`。
