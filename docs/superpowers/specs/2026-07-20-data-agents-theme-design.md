# 「数据智能体」主题设计

## 目标

在 `projects/`（项目实践）下新增第四个分类「数据智能体」，包含三篇**递进式**项目写作稿，共享一套电商订单数据集，串成一条「对话式数据分析」链路。三个方向：数据 AI 自动标签化、Text-to-SQL、数据可视化。

## 决策摘要

| 维度 | 决策 |
|------|------|
| 组织方式 | 串成一个「数据智能体」主题（非三个独立项目） |
| 共享场景 | 电商订单数据集 |
| 技术栈 | 统一 TypeScript/Node（LangChain.js + Vercel AI SDK；可视化用 React/ECharts） |
| 内容形态 | 文档写作稿，同现有项目风格；**不建可运行代码 / demo / 真实数据库** |
| 模块顺序 | 按数据流水线：标签化 → Text-to-SQL → 可视化 |

## 共享数据集（电商订单）

在 `index.md` 中定义一次，后续三篇统一引用，体现"复用一套数据"的主线。约定四张表：

| 表 | 说明 | 关键字段 |
|----|------|----------|
| `users` | 用户 | id, name, city, register_date |
| `products` | 商品 | id, name, category, price |
| `orders` | 订单 | id, user_id, product_id, quantity, amount, created_at |
| `reviews` | 评价 | id, product_id, user_id, content, rating, created_at |

数据流转关系：`reviews.content`（原始文本）由标签化模块打标 → `orders`/`products` 由 Text-to-SQL 查询 → 查询结果由可视化模块呈现。

## 每篇内容结构

统一沿用现有项目模板：**项目概述（功能范围 + 技术栈）→ 架构设计（ASCII 图）→ 核心功能（代码片段）→ 关键技术点 → 实现步骤 → 进阶功能**。

### 01 数据 AI 自动标签化（理数据）

- 定位：原始数据（此处为 `reviews.content`）→ LLM 按预定义 taxonomy 分类打标 → 结构化输出。
- 关键技术点：
  - 标签体系（taxonomy）设计：预定义标签 vs 自动发现。
  - 结构化输出约束：zod schema / function calling，保证输出可解析、字段稳定。
  - 批量处理与成本控制。
  - 置信度 + human-in-the-loop 复核回环。
- 落地示例：给电商 `reviews` 打「情感（正/负/中）+ 问题分类（物流/质量/客服/价格）」标签，写回结构化字段。
- 进阶：主动学习（挑不确定样本给人标）、few-shot 自举、标注一致性评估。

### 02 Text-to-SQL（问数据）

- 定位：自然语言 → SQL → 执行 → 把结果读回成自然语言。
- 关键技术点：
  - schema / 字段含义注入到 prompt。
  - few-shot 示例引导 SQL 风格。
  - **执行报错自纠错回环**：SQL 执行失败时把错误回喂给模型重试。
  - 安全约束：只读、防 SQL 注入。
  - 结果回读：把查询结果总结成自然语言。
  - 模糊问题的多轮澄清（可作为进阶）。
- 落地示例：查 `orders`/`products`，如"上个月各品类销售额 Top 5"。
- 进阶：RAG 检索相关表/历史查询、语义层/指标定义、多轮澄清。

### 03 数据可视化（看数据）

- 定位：自然语言 + 数据 → 判断图表类型 → 生成结构化图表 spec → 渲染。
- 关键技术点：
  - 图表选型：按数据维度/意图选型（时间趋势→折线、占比→饼/柱、分布→散点）。
  - 生成**结构化图表 spec**（Vega-Lite / ECharts option）而非让 LLM 直接画图。
  - 字段映射（x/y/维度/度量）。
  - 图表解读（narrative）：为图表配一段自然语言说明。
  - 可读性 / 配色。
- 落地示例：把 Text-to-SQL 的查询结果自动画成图；React + ECharts 渲染示例代码。
- 进阶：多图 dashboard 生成、异常高亮、图表解读自动化。
